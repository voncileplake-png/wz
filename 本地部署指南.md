# 本地部署指南

本指南将帮助您在本地部署和运行酒店网站项目。

## 系统要求

- **Node.js**: v18.17 或更高版本（当前已安装: v22.17.1 ✓）
- **npm**: v10.9.2 或更高版本（当前已安装: 10.9.2 ✓）
- **操作系统**: Windows / macOS / Linux

## 部署步骤

### 方法一：开发模式运行（推荐用于开发和测试）

开发模式会自动监听文件变化并热重载，适合开发调试。

#### 1. 安装依赖（如果尚未安装）

```bash
npm install
```

或使用 pnpm（如果已安装）：

```bash
pnpm install
```

#### 2. 启动开发服务器

```bash
npm run dev
```

服务器将在 `http://localhost:3000` 启动。

#### 3. 访问网站

在浏览器中打开：`http://localhost:3000`

### 方法二：生产模式构建和运行（推荐用于生产环境）

生产模式会先构建优化后的代码，然后运行生产服务器。

#### 1. 构建项目

```bash
npm run build
```

这将生成优化的生产版本，构建输出保存在 `.next` 目录中。

#### 2. 启动生产服务器

```bash
npm start
```

服务器将在 `http://localhost:3000` 启动。

#### 3. 访问网站

在浏览器中打开：`http://localhost:3000`

## 可用脚本命令

- `npm run dev` - 启动开发服务器（端口 3000）
- `npm run build` - 构建生产版本
- `npm start` - 启动生产服务器
- `npm run lint` - 运行代码检查

## 配置说明

- **Next.js 配置**: `next.config.mjs`
  - 图片优化已启用
  - 压缩已启用
  - 字体优化已启用

- **端口配置**: 默认端口为 3000
  - 如需更改端口，使用：`npm run dev -- -p <端口号>`
  - 例如：`npm run dev -- -p 3001`

## 常见问题

### 1. 端口已被占用

如果 3000 端口已被占用，可以指定其他端口：

```bash
npm run dev -- -p 3001
```

### 2. 依赖安装失败

如果遇到依赖安装问题，可以尝试：

```bash
# 清理缓存
npm cache clean --force

# 删除 node_modules 和重新安装
rm -rf node_modules package-lock.json
npm install
```

### 3. 构建错误

如果构建时出现错误，检查：

- Node.js 版本是否符合要求
- 所有依赖是否已正确安装
- TypeScript 类型错误（项目配置为忽略构建错误）

## 停止服务器

在运行服务器的终端中按 `Ctrl + C` 即可停止服务器。

## 项目结构

```
酒店网站文件/
├── app/              # Next.js App Router 页面
├── components/       # React 组件
├── hooks/           # 自定义 React Hooks
├── lib/             # 工具函数和库
├── public/          # 静态资源文件
├── styles/          # 样式文件
├── package.json     # 项目配置和依赖
└── next.config.mjs  # Next.js 配置
```

## 技术支持

如遇到问题，请检查：
1. Node.js 和 npm 版本是否正确
2. 依赖是否完整安装
3. 端口是否被占用
4. 防火墙是否阻止了端口访问
